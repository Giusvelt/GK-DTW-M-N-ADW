# Workflow: Backlog Manager

Questo workflow serve a garantire che nessuna idea o richiesta importante vada persa quando si cambia focus durante lo sviluppo.

## Passaggi:

1.  **Identificazione**: Quando una richiesta dell'utente viene posticipata per dare priorità ad altro, inseriscila immediatamente nel file `C:\Users\giuse\Desktop\ANTIGRAVITY\geokanban_v3\.agent\backlog.md`.
2.  **Dettagli**: Per ogni task, specifica:
    - **Titolo**: Breve e chiaro.
    - **Contesto**: Perché è stato tralasciato.
    - **Dependency**: Se bloccato da altri task.
3.  **Cross-Reference**: Aggiungi una nota in `task.md` nella sezione `[Postponed]` puntando al backlog.
4.  **Recupero**: All'inizio di ogni nuova chat, leggi il file `backlog.md` e proponi all'utente di selezionare uno dei task pendenti.
